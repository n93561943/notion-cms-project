# PRD: Notion CMS 기반 개인 기술 블로그

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 개인 개발 블로그 |
| **목적** | Notion을 CMS로 활용한 개인 기술 블로그 구축 |
| **핵심 가치** | Notion에서 글을 작성하면 자동으로 블로그에 반영되는 워크플로우 |

### CMS로 Notion을 선택한 이유

- **익숙한 편집 환경** — 별도 관리자 페이지 없이 Notion에서 바로 글 작성·편집 가능
- **구조화된 데이터** — 데이터베이스 속성(카테고리, 태그, 상태 등)으로 메타데이터 관리
- **비용 절감** — 별도 DB/CMS 서버 없이 Notion API만으로 콘텐츠 제공
- **빠른 MVP 구축** — 콘텐츠 관리 로직을 직접 구현할 필요 없음

## 4. Notion 데이터베이스 구조

블로그 콘텐츠를 관리하는 Notion 데이터베이스의 속성 정의:

| 속성명 | Notion 타입 | 설명 | 필수 |
|--------|------------|------|------|
| **Title** | `title` | 글 제목 | ✅ |
| **Category** | `select` | 카테고리 (예: React, Next.js, TypeScript 등) | ✅ |
| **Tags** | `multi_select` | 태그 목록 | — |
| **Published** | `date` | 발행일 | ✅ |
| **Status** | `select` | 상태 — `초안` / `발행됨` | ✅ |
| **Content** | page content | 본문 (Notion 페이지 블록) | ✅ |

> **참고**: 블로그에는 `Status`가 `발행됨`인 글만 노출한다.

## 6. 화면 설계

### 6.1 홈 (`/`)

- **구성**: 헤더, 검색바, 카테고리 필터, 글 목록 그리드, 푸터
- **글 카드 정보**: 제목, 카테고리 뱃지, 태그 목록, 발행일
- **정렬**: 발행일 최신순
- **레이아웃**: 데스크톱 3열 → 태블릿 2열 → 모바일 1열 그리드

### 6.2 글 상세 (`/posts/[slug]`)

- **구성**: 헤더, 글 메타 정보, 본문 콘텐츠, 푸터
- **메타 정보**: 제목, 카테고리, 태그, 발행일
- **본문**: Notion 블록을 HTML로 변환하여 렌더링
- **내비게이션**: 상단에 "← 목록으로" 링크

### 6.3 카테고리 (`/categories/[category]`)

- **구성**: 헤더, 카테고리명, 해당 카테고리 글 목록, 푸터
- **레이아웃**: 홈과 동일한 글 카드 그리드

## 7. API 설계

### 7.1 Notion API 연동 방식

- **서버 사이드 전용** — Notion API 키는 서버에서만 사용 (환경 변수: `NOTION_API_KEY`, `NOTION_DATABASE_ID`)
- **서버 컴포넌트에서 직접 호출** — Next.js App Router의 서버 컴포넌트에서 Notion API 호출
- **캐싱** — Next.js의 `fetch` 캐싱 또는 `revalidate` 옵션으로 적절한 ISR 적용

### 7.2 주요 API 함수

| 함수명 | 설명 | Notion API |
|--------|------|-----------|
| `getPosts()` | 발행된 글 목록 조회 (Status=발행됨, 최신순 정렬) | `databases.query` |
| `getPostBySlug(slug)` | 개별 글 메타 정보 조회 | `databases.query` (필터) |
| `getPostContent(pageId)` | 글 본문 블록 조회 | `blocks.children.list` |
| `getCategories()` | 카테고리 목록 조회 | `databases.query` → 고유 카테고리 추출 |
| `getPostsByCategory(category)` | 카테고리별 글 목록 조회 | `databases.query` (필터) |

### 7.3 환경 변수

```env
NOTION_API_KEY=secret_xxx        # Notion 내부 통합 API 키
NOTION_DATABASE_ID=xxx           # 블로그 데이터베이스 ID
```
